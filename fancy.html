You will need the following

  • USD3 (or regional equivalent) in your Nintendo eShop account.
  • A stock 3DS or 2DS, old or new.

Optionally, this may help

  • A computer with a GPU permitting use of 4GB or more of video memory (VRAM).
  • Python 3.6 (preferably) or 2.7.
  • Another person to assist with their modified console for seedstarter.

Section 2 — Initial steps
If you have to, go into Windows Explorer's options and disable hiding of known file extensions. As it is beyond the scope of this guide, search for this information yourself.

After having fulfilled everything in the above section, only then are you ready to begin without encountering any future difficulties. Unlike other guides online, this one provides a single archive providing a universal set of files up to 11.8.

On the console

  1. Open the Nintendo eShop.
  2. Purchase an injectable game from this list.
    • Keep that page open; Information from it may help you later.
    • If you aren't sure, purchase SUDOKU. The file generated with the process below will be 4B344445.bin (USA) or 4B344456.bin (EUR). There is no SUDOKU equivalent for JPN but any of the listed games for that region will work.
  3. Exit the eShop and open System Settings; Go to Data Management, DSiWare.
  4. Find your compatible title and copy it onto the SD card. Permit this operation in the following prompt.
  5. Power off the Nintendo 3DS and remove the SD card.
    • If you own a console with the "new" wordmark, you can do one better: Go into microSD Management and connect with your PC instead.

On the computer

  1. Insert the 3DS' SD card into another device which permits you to manipulate files directly (PC, tablet, smartphone etc.).
  2. Download this archived suite of files. (seedkit.zip)
    • All version-agnostic files in this archive were obtained from 3ds.hacks.guide and have already been assembled in the correct fashion for execution on the 3DS.
  3. Extract the contents of the archive onto the root of the SD card.
    • This means all directories and files should be alongside Nintendo 3DS, not inside of it, and especially not in its own directory.
  4. Download the following additional files depending on your firmware version:
    • 11.4 - 11.7: B9STool v4.0.0
      • If you already have a compatible DSiWare title, do not update the system firmware! Any future instructions to update should be ignored.
    • 11.8 onward: B9STool v4.0.2
      • v4.0.2 is not compatible with 11.7 or less!
      • If you didn't have a compatible DSiWare title already, you may be forced to update the system firmware which makes this the only option.
  5. Navigate to Nintendo 3DS; you should see two directories.
    • If you see more than two, search for the filename of the game you had copied, per the list you may had previously obtained it from. Results should show one entry in Nintendo 3DS/[id0]/[id1]/Nintendo DSiWare.
  6. Copy your id0.
    • It is the name of the 32-character directory alongside Private.
    • Example: 4a5b6c7d8effe0d1c2b3aa9b0c1d2e3f
    • You can use your file manager's rename function to copy it. Keep it somewhere in case you need to copy it multiple times.

Does the 3DS' SD card lack space? Expand and read this.

There are three ways to handle a lack of disk space on the SD card. Before that, a brief of how the 3DS handles a new card:

When the console senses there is no data of its own on the SD card, it will write a new Nintendo 3DS directory. This is important to understand, as its contents will later be used to modify your console. It also immensely simplifies things as when finding what is known as your ID0 for later use with seedminer, you only have one directory which can be it, thus there is no need to search for the DSiWare title you'll copy in later steps to confirm this information.

Understanding this, here are the three ways to handle an SD card which is near its maximum capacity;

  • Do nothing

    This is a foolish thing to do if you are uncertain of the space required. You should have, as a reasonable minimum 1.4GB available. If you lack this space, see next item.

  • Move everything to other media

    Moving the entire Nintendo 3DS directory out of the SD card will certainly yield much more disk space for modding, depending on how heavily you relied upon the woefully-undersized SD card which comes with most consoles out of the box. After you are finished performing steps in the following sections of this guide, including the creation of a system memory backup then you can move the transferred Nintendo 3DS directory to your SD card and merge all contents, should there still be room. If you find yourself unable to, or are anticipating purchase of a larger SD card, see next item.

  • Move everything to a larger (micro)SD card

    Had you already followed the previous item's advice, it's simply a matter of transferring all contents from the original SD card into a new one. For larger cards, third-party Windows disk management tools or a Linux / BSD live session with GParted / KParted / Gnome disk utility can enable writing a FAT32 partition on media WIndows refuses to write it for, despite its 2TB maximum size. If you are anticipating upgrade to a "New" 3DS and are still on an "Old" model with the more accessible full-size SD card, upgrading to a higher-capacity microSD card will futureproof the contents therein, as you can follow these steps to hack your new console and still use most of what's on the SD card, except maybe CIA-installed utilities you will have to re-acquire from FBI's titleDB elsewhere on the Internet.

    In the rare case you could not move your transferred Nintendo 3DS directory, when migrating to a new SD card, move the contents of the previous SD card used to mod your console first before moving the transferred Nintendo 3DS directory. This will ensure previous system preferences are transferred, overwriting newer files which were created as a response of nothing being in the SD card.

You may notice other directories, such as DCIM. This is just pictures and such. If there is a lot of content there, move that out of the SD card also. If you have an MP3 directory or other directories with music in them for Nintendo 3DS Sound, those contents may interfere with the process of modding as that may also take a significant amount of disk space. Should there be more than the Nintendo 3DS directory you need to move, creating a new directory on a larger media before writing to represent the present contents of the 3DS' SD card before modding will make returning the full contents of the SD card into its original place much easier.

With the above information, you should be able to handle the contents of the 3DS' SD card with confidence knowing you will lose nothing you wish to keep.

Section 3 — Seedminer

Can't use Seedminer? No problem! Expand this advice to learn more.

You can use Bruteforce Movable instead of following most of this section. To use this service:

  1. Obtain your part1 from somebody else with a modified console
    • While you can try to obtain your part1 using this service, rumour claims it doesn't always yield good results.
  2. Provide your console's id0
    • Your id0 was obtained earlier; you should still have it per instructions in section 2.
  3. Press Start and wait.

You may be waiting a long while for this depending on queue (shown in the blue heading on top), so if you have an nVidia GTX 960 or equivalent, try mining your own seed instead!

If you do not have a good GPU (anything around 2008) inside your machine by nVidia, AMD or an AIB using either company's hardware, you may not be able to mine your seed quickly.

This process is necessary for decrypting a special file on the 3DS later used with a service for compiling your CFW entrypoint. Without it, you may not be able to install CFW into the 3DS without using a different method.

Requirements for installing and using seedminer.

  • Python 3.7 (preferred) or 2.7, from Python Software Foundation's landing page.
  • Seedminer itself.
  • Your console's movable_part1.sed (hereon: part1), detailed below.
  • The console's first 32 characters of its SHA-256 hash (hereon: id0), detailed below.

Obtaining your part1

These instructions require another person with a modified console to assist you. Perform the following with whoever is helping you:

  1. Request and add their friend code
    • Seedstarter does not work with provisionally added friends: Both sides must add one another.
  2. Have instaled on their SD card Seedstarter from the seedminer Github repository, if applicable.
    • Alternatively, they can obtain Seedstarter from FBI's TitleDB menu. (Though, it may show up as seedminer there.)
  3. Wait for them to open seedstarter and perofrm the following steps.
    1. Press B, A, then Start in seedstarter.
    2. After obtaining all friend's seeds, access the seedstarter directory on their SD card.
    3. Find the .sed file with the friend code of the user they are assisting and rename it to movable_part1.sed.
      • Example: 0_1234_4567_7890.sed becomes movable_part1.sed for use with whoever has friend code 1234-4567-7890.
    4. Provide to you movable_part1.sed.
  4. Make use of movable_part1.sed for either:
    • Mining on your own PC.
    • Have your assistant mine for you on their PC.

Mining your seed

If Bruteforce Movable is down and somebody else needs to mine for you, your id0 needs to be provided to them for a successful operation.

If you only have Python 2.7, expand this to view command substitutions.
  • Instead of seedminer_launcher3.py, seedminer_launcher.py is used.
  • If py -2 shorthand is not an option, use %systemdrive%\Python27\python.exe instead.
  1. Locate movable_part1.sed; Where seedminer was extracted, navigate to where seedminer_launcher3.py is and move it there.
  2. Hold Right Shift while opening the context menu to access additional options; choose to open command prompt / powershell.
    • This does not mean to execute the Python shell environment! The remaining instructions are performed outside of an executed command-line program.
  3. Execute py -3 seedminer_launcher3.py update-db.
  4. Execute py -3 seedminer_launcher3.py id0 [id0].
    • Example: py -3 seedminer_launcher3.py id0 4a5b6c7d8effe0d1c2b3aa9b0c1d2e3f.
    • If you are confused about what your id0 is, see section 2.
  5. Execute py -3 seedminer_launcher3.py gpu.
    • cpu is a possible argument as well. Also possible: You waiting up to a day or more for CPU mining.
    • If errors did occur, try opening cmd as an administrator, rinse and repeat for multiple seeds.
  6. If nothing produced fatal errors, multiple cmd instances will open: Do not close them.
  7. Obtain your movable.sed from the directory seedminer was executed in.

Section 4 — DSiWare injection
None of these steps are of any use without your movable.sed. Acquire it from Bruteforce Movable or make it yourself in seedminer (see section 3).

These series of steps are necessary for producing your CFW entrypoint. Without it, the additional software on your SD card is rendered useless.

At your computer

  1. Go to this website to build your patched bin.
  2. The fields shown refer to the following:
    • Unique identifier for the patched .bin file.
    • Region of your console (usually shown on your console).
    • Location of your copied DSiWare .bin.
    • Location of your movable.sed.
    • You can skip E-Mail: Build will not take long.
  3. Hit the big blue Build button and watch the left side for your build to complete.
    • 🔵 means build completed with a usable artifact.
    • 🔴 means the movable.sed or .bin file was bad and needs to be remade.
  4. If you see your build is successful, click it and download the artifact made onto your SD card in Nintendo DSiWare.
  5. Rename, move or delete the the original, untouched .bin file.
  6. Rename the .patched file so it ends in .bin instead.
    • The previous three steps are intended to overwrite your game with a modified copy which launches b9stool.
  7. Properly eject the SD card from your PC and put it in the console.

At your console

  1. Open System Settings, Data Management, DSiWare.
  2. In the SD Card tab, choose to copy the patched bin to System Memory. Permit this operation in the following prompt.
  3. When finished, exit to Home Menu and open the patched game.

  4. Press B to back out until you see the console's IP address.

With the IP address noted, you can begin installing utilities on the PC which can utilize this information to connect with the console.

Configuring and use of installed software

Input Redirection

  1. Open Input Redirection Client.
  2. Enable Input Redirection on the 3DS.
  3. Give it your console's IP address.
    • If you prefer button placement to match the console, enable the boxes for inverting the face buttons.

No XInput controller? No problem! Expand to see why.

If you lack an XInput-compatible controller such as those for Xbox 360 or Xbox One to use with the redirection client shown above, you're in luck; there exists software which can make any controller compatible.

  1. Download the 32-bit version of DacoTaco's Xbox 360 controller emulator (hereon: x360ce).
  2. Extract, then run the software where it had been extracted.
    • You can also extract x360ce into the directory whose program you want to use it with.
  3. Open x360ce, then permit it to find a functional preset for your controller.
  4. If that doesn't work straight away, end its task, then select Auto in the General tab.
  5. Alternatively, look in the Controller Settings tab to find a functional preset matching your hardware.
  6. If applicable, copy files x360ce.ini and xinput1_3.dll where InputReciredtionClient-Qt.exe is.
  7. Rename xinput1_3.dll to xinput1_4.exe.
    • Alternatively, create a hardlink for xinput1_3.dll named xinput1_4.dll if the software resides on an NTFS partition.
  8. Run InputRedirectionClient-Qt.exe. If successful, you should hear a tone to confirm x360ce is emulating XInput.

Video casting

Some items may simply not work, and launching certain titles while casting may crash the console.
Crashes will not permanently disable the console; simply hold down the power button until the lights go out and power it back on.
  1. On the 3DS, open BootNTR selector.
  2. Go along with default everything, and select version 3.6.
  3. Return to home menu.
  4. Launch Snickerstream.
    • In doing this, a new settings.ini file is made where the executable exists. These settings may be of interest:
      • IpAddr: The default console IP address Snickerstream will look for.
      • Framelimit: Add this in a new line to specify the maximum number of casted frames.
      • notepad.exe cannot read the line breaks in this file, use an alternative editor such as Notepad++.
  5. Give Snickerstream the console's IP address. (It's the same one you give to Input Redirection Client.)
  6. In the Preset section, specify Best framerate to test, initially.
  7. Choose to connect; If it worked, scale video with [Up] and [Down] on the keyboard.
  8. When finished streaming, press [Esc] to exit.
